# План исправления проблемных формул Google Sheets

## Цель
Исправить только те записи, которые по результатам аудита оказались фактически неверными, внутренне противоречивыми или вводящими пользователя в заблуждение.

План не расширяет scope до полного выравнивания всех 80 сигнатур с документацией Google. Подход будет минимальным: починить проблемные формулы и связанные с ними локализованные данные, не трогая формулы, которые лишь слегка упрощены, но не ломают смысл.

## Файлы, которые потребуется изменить
- `src/data/formulas.js`
- `src/data/localizedFormulas.js`

## Что именно исправлять

### 1. Исправить фактически неверные или противоречивые записи в `src/data/formulas.js`
Обязательный список:
- `IFERROR`
  - расширить `description`, чтобы функция была описана как обработчик любых ошибок, а не только `#N/A` / `#VALUE!`;
- `HLOOKUP`
  - сделать `example` и `EXAMPLE_NARRATIVES` внутренне согласованными, чтобы lookup-key не лежал внутри того же стартового адреса, который narrative описывает как начало таблицы;
- `XLOOKUP`
  - обновить `syntax`, чтобы были отражены `missing_value`, `match_mode`, `search_mode`;
- `LOOKUP`
  - переписать `description`, `syntax` и `EXAMPLE_NARRATIVES`, чтобы явно показать approximate-match-поведение и зависимость от отсортированного диапазона;
- `QUERY`
  - добавить `[headers]` в `syntax` и привести пример к более предсказуемому виду;
- `SUMIFS`
  - заменить локалезависимый date criterion на переносимый вариант через `DATE(...)` и конкатенацию условия;
- `SEQUENCE`
  - привести `syntax` в соответствие с реальным набором аргументов, не ломая текущий простой пример;
- `WRAPROWS`
  - добавить `[pad_with]` в `syntax` и обновить пример/narrative так, чтобы результат не противоречил фактическому поведению по умолчанию;
- `WRAPCOLS`
  - сделать то же самое, что для `WRAPROWS`;
- `GOOGLEFINANCE`
  - расширить `syntax`, сменить пример на более реалистичный, а narrative сузить до реально возвращаемого результата вместо формулировки про «сопутствующие поля».

Точечные смысловые уточнения, которые стоит сделать в том же проходе:
- `IFS`
  - в `description` или `result` явно отразить, что без совпадения функция может вернуть `#N/A`;
- `SWITCH`
  - отразить optional default / `#N/A`, если вариант не найден;
- `ISBLANK`
  - убрать слишком общее описание «пустая ли ячейка» и уточнить поведение для реально пустой ссылки.

### 2. Синхронно обновить локализованный слой `src/data/localizedFormulas.js`
Исправления в `formulas.js` не покроют полностью `en` и `ua`, потому что в локализованном слое есть отдельные:
- `FORMULA_DESCRIPTIONS`
- `FORMULA_EXAMPLE_OVERRIDES`
- `FORMULA_EXAMPLE_NARRATIVES`

Поэтому нужно синхронно обновить:
- локализованные `IFERROR` descriptions;
- локализованные descriptions для `IFS`, `SWITCH`, `ISBLANK`, если русская формулировка будет уточнена по смыслу;
- `SUMIFS` example overrides для `en` и `ua` на locale-safe вариант с `DATE(...)`;
- локализованные narratives для:
  - `HLOOKUP`
  - `LOOKUP`
  - `QUERY`
  - `WRAPROWS`
  - `WRAPCOLS`
  - `GOOGLEFINANCE`
- при необходимости narratives для `XLOOKUP`, `IFS`, `SWITCH`, `ISBLANK`, если после правок в русском слое они останутся смыслово расходиться.

### 3. Починить подсказки аргументов для карточек
После расширения `syntax` автогенерация `exampleDetails.variables` начнёт использовать новые имена аргументов. Чтобы UI не показывал generic-текст `рабочий аргумент функции`, нужно обновить словари аргументов в обоих файлах.

Обязательные новые ключи для текущего scope:
- `missing_value`
- `match_mode`
- `search_mode`
- `headers`
- `columns`
- `step`
- `pad_with`
- `attribute`

Низкозатратные исправления, которые имеет смысл включить сразу, потому что они уже сейчас деградируют в generic-label:
- `criterion`
- `criteria_range1`
- `criterion1`
- `criteria_range2`
- `criterion2`
- `sum_range`
- `places`
- `search_for`
- `replace_with`
- `occurrence_number`
- `regular_expression`
- `value2`
- `value_if_na`

Эти ключи нужно добавить:
- в `ARGUMENT_HINTS` внутри `src/data/formulas.js`;
- в `ARGUMENT_HINTS.ru`, `ARGUMENT_HINTS.en`, `ARGUMENT_HINTS.ua` внутри `src/data/localizedFormulas.js`.

## Принципы правок
- Сохранять изменения минимальными и локальными.
- Не переписывать весь датасет ради полной энциклопедичности.
- Для проблемных функций предпочитать точную, но всё ещё компактную сигнатуру.
- Для примеров выбирать переносимые формулы, которые не зависят от `dd.mm.yyyy` и не создают ложное ощущение guaranteed-behavior там, где Google Sheets ведёт себя условно.
- Для `IMPORT*`/внешних функций в этом проходе не делать широкую переработку placeholder-примеров, если они не входят в список обязательных исправлений выше. Это отдельный cleanup, чтобы не раздувать scope.

## Порядок выполнения
1. Внести правки в `baseFormulas` внутри `src/data/formulas.js`.
2. Обновить соответствующие записи в `EXAMPLE_NARRATIVES` в том же файле.
3. Расширить `ARGUMENT_HINTS` в `src/data/formulas.js` под новые и уже проблемные токены.
4. Внести зеркальные правки в `src/data/localizedFormulas.js`:
   - descriptions;
   - example overrides;
   - localized narratives;
   - локализованные словари аргументов.
5. Повторно перечитать затронутые записи и проверить, что `syntax`, `example`, `description` и `exampleDetails` не расходятся по смыслу.

## Проверки после изменений
Обязательные:
- `npm run lint`

Желательные для дополнительной уверенности после правок в двух больших data-файлах:
- `npm run build`

Ручная проверка по коду:
- убедиться, что `Card` продолжает безопасно рендерить `formula.syntax`, `formula.example` и `exampleDetails.variables` без пустых или generic-аргументов для исправленных формул;
- выборочно перечитать исправленные записи в `ru`, `en`, `ua` и проверить, что смысл синхронизирован.

## Ожидаемый результат
После выполнения этого плана:
- карточки проблемных формул перестанут давать фактически ложные ожидания;
- примеры станут переносимее между локалями;
- `Variables` для затронутых формул перестанут сваливаться в generic-подсказки;
- русская, английская и украинская версии перестанут расходиться на исправленных формулах.

## Явно вне scope этого прохода
- полный пересмотр всех сокращённых сигнатур, которые сейчас просто короче документации, но не искажают поведение;
- замена всех placeholder URL-примеров на гарантированно рабочие публичные источники;
- изменение UI-логики копирования формул под разделители аргументов конкретной локали.
