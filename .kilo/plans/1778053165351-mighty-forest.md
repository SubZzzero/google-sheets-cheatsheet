# Анализ проекта и рабочий контекст

## Статус
- Режим: plan mode / read-only.
- Единственный разрешённый для записи файл: этот план.
- Дополнительные вспомогательные файлы для памяти пока не создавались, потому что текущий режим запрещает запись куда-либо ещё.
- Этот файл используется как временный consolidated context для дальнейшей работы.

## Краткое резюме проекта
- Проект: `google-sheets-cheatsheet`.
- Тип: небольшой SPA на React + Vite.
- Назначение: каталог-шпаргалка по формулам Google Sheets с категориями, карточками, синтаксисом и расширенными примерами.
- Язык интерфейса: в основном русский.
- Деплой: Firebase Hosting.
- Текущее состояние git: рабочее дерево выглядит чистым по `git status --short`.
- Текущая ветка: `main`.

## Технологический стек
- Runtime/UI: `react 19`, `react-dom 19`.
- Сборка: `vite 8`.
- Линтинг: `eslint 9`, `@eslint/js`, `eslint-plugin-react-hooks`, `eslint-plugin-react-refresh`.
- Язык: JavaScript / JSX, TypeScript отсутствует.
- Стили: CSS Modules + 2 глобальных CSS-файла (`reset.css`, `style.css`).
- Хостинг: `firebase.json`, `.firebaserc`.

## Архитектура приложения
### Точка входа
- `src/main.jsx` рендерит `<App />` в `#root`.
- Роутинга нет, приложение одностраничное и держит состояние внутри `App`.

### Корневой контейнер
`src/App.jsx` отвечает за:
- импорт формул из `src/data/formulas.js`
- построение списка категорий
- сортировку категорий через `CATEGORY_ORDER`
- подсчёт количества формул в каждой категории
- хранение состояния:
  - `activeCategory`
  - `isSidebarOpen`
- переключение между домашним экраном и экраном карточек выбранной категории

### UI-поток
Логика отображения:
1. Если `activeCategory === null`, показывается `Home`.
2. После выбора категории показываются:
   - `Sidebar` со списком категорий
   - `CardList` с карточками формул выбранной категории
3. `Header` показывает кнопку меню только когда категория уже выбрана.
4. `Footer` всегда отображается.

### Компоненты
- `Header`: sticky-шапка, логотип, название, мобильная кнопка меню.
- `Sidebar`: список категорий; на мобильных превращается в slide-in панель с backdrop.
- `Home`: hero-блок, инфо-карточки, grid категорий с описаниями и счетчиком формул.
- `CardList`: заголовок категории, пояснение, кнопка возврата на старт, grid карточек.
- `Card`: отдельная формула, копирование синтаксиса, описание, блок `Example`.
- `Footer`: ссылка на GitHub.

## Данные
### Главный источник данных
- `src/data/formulas.js` — центральный и самый важный файл проекта.
- Размер файла большой: 1091 строка.
- Внутри нет внешних запросов или API; проект целиком опирается на локальный массив данных.

### Состав данных
В `baseFormulas` сейчас 80 формул:
- `Базовые` — 10
- `Часто используемые` — 10
- `Логические` — 10
- `Поиск` — 10
- `Текст` — 10
- `Данные` — 4
- `Массивы` — 6
- `Дата и время` — 10
- `Импорт` — 10

### Форма одной записи
Каждая формула содержит:
- `id`
- `name`
- `category`
- `description`
- `syntax`
- `example`

### Обогащение данных
Файл не просто хранит массив. В конце есть логика построения производных данных:
- `ARGUMENT_HINTS` — словарь пояснений для аргументов формул.
- `EXAMPLE_NARRATIVES` — словарь narrative-описаний для примеров.
- `extractSyntaxArgs(syntax)` — извлечение аргументов из строки синтаксиса.
- `buildVariablesLegend(syntax)` — построение списка "аргумент — пояснение".
- `buildExampleDetails(formula)` — формирование блока `purpose / variables / calculation / result`.
- `export const formulas = baseFormulas.map(...)` — итоговый экспорт с `exampleDetails`.

### Важный вывод
- `formulas.js` — это одновременно контент, словарь пояснений и слой нормализации/derivation.
- Любые изменения в карточках почти наверняка затронут этот файл или контракт его экспорта.

## UI и визуальная система
### Общий стиль
- Светлая тема.
- Основной визуальный язык: белые карточки, мягкие тени, серо-зелёная палитра, rounded corners.
- Шрифт: `Inter`, далее системные sans-serif fallback.
- Основные глобальные цвета заданы не через design tokens, а напрямую hex-значениями в CSS-модулях.

### Layout
- Корневой layout: `App.module.css`.
- Desktop: grid `280px + content`.
- Home screen использует single-column layout.
- На `< 900px` layout переключается в блочный режим, sidebar становится overlay.

### Наблюдения по UI
- Домашний экран уже переработан и выглядит более продуманным, чем “чистый список”.
- В интерфейсе есть смешение языков:
  - русский контент почти везде
  - английские подписи `COPY`, `COPIED`, `Example`
- `Home` использует emoji-иконки в `CATEGORY_META`.

## Качество и риски
### Технические риски
- `src/data/formulas.js` очень крупный и монолитный: будущие изменения в данных будут дорогими и шумными в diff.
- Данные, пояснения аргументов и narrative-логика слиты в одном файле.
- Нет тестов на корректность структуры формул и генерации `exampleDetails`.
- Нет runtime-защит от неполных/битых записей формул.

### UX/контентные риски
- `index.html` выглядит неактуально для production UI:
  - `lang="en"`, хотя интерфейс русский
  - `title` технический: `google-sheets-cheatsheet`
  - `favicon` указывает на `/favicon.svg`, но в корневом обзоре такой файл не был найден
- Смешение русского и английского в интерфейсе может выглядеть как незавершённая локализация.
- Кнопка копирования зависит от `navigator.clipboard`; fallback-поведение минимальное.

### Поддерживаемость
- Проект маленький и понятный, поэтому быстрые изменения делать легко.
- Самая важная зона для аккуратной работы: `formulas.js`.

## Предлагаемый рабочий план на фазу реализации
1. Определить ближайшую пользовательскую цель: UI, контент, навигация, SEO, refactor данных или качество.
2. Перед изменениями повторно проверить, какие из ключевых файлов реально затрагиваются.
3. Если задача касается данных, сначала стабилизировать контракт `formula` и только потом менять представление.
4. Если задача касается интерфейса, держать совместимость desktop/mobile, потому что sidebar и layout уже имеют адаптивную логику.
5. После изменений прогонять минимум `npm run lint`, а при UI-изменениях ещё и `npm run build`.
6. Если начнётся масштабная работа с контентом, рассмотреть разбиение `formulas.js` на модули по категориям.
