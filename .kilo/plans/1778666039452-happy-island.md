# Plan: Add ScrollToTop

## Goal
Add a neat `ScrollToTop` control that appears only after the user scrolls down, works across the current app screens, matches the existing UI, and remains accessible.

## Current Findings
- `src/App.jsx` owns the top-level screen state and conditionally renders `Home` or `CardList`.
- The app uses normal window scrolling; there is no custom scroll container.
- The header is sticky (`z-index: 30`), and the mobile sidebar/backdrop use `z-index: 40` / `35`.
- The project keeps component styling in CSS Modules and UI text in `src/i18n.js`.
- Global focus styling already exists in `src/assets/style.css`.

## Implementation Plan
1. Add a new component `src/components/ScrollToTop/ScrollToTop.jsx`.
   - Keep its own internal visibility state.
   - Subscribe to `window` scroll events in an effect.
   - Show the button only after a reasonable scroll threshold (about 300px).
   - On click, call `window.scrollTo({ top: 0, behavior })`.
   - Respect reduced motion by switching to non-smooth scrolling when `prefers-reduced-motion: reduce` is enabled.
   - Render a semantic `<button type="button">` with a localized `aria-label`.

2. Add `src/components/ScrollToTop/ScrollToTop.module.css`.
   - Use `position: fixed` with bottom/right offsets.
   - Match the current visual language: white surface, soft border, subtle shadow, rounded corners.
   - Provide hover and `:focus-visible` friendly behavior.
   - Keep the button compact and non-intrusive on mobile.
   - Use a `z-index` high enough to sit above content, but below the mobile sidebar/backdrop so the drawer remains dominant when open.

3. Mount the component once in `src/App.jsx`.
   - Import and render it at the app-shell level, near the main content/footer.
   - Pass localized UI text from the existing `ui` object.
   - Avoid wiring it into `Home` or `CardList`, since the control should be global and not duplicated.

4. Extend `src/i18n.js` with button text for all supported locales.
   - Add a small `scrollToTop` section or equivalent app-level UI field.
   - Minimum string needed: accessible label.
   - Suggested values:
     - `en`: `Scroll to top`
     - `ua`: `Прокрутити вгору`
     - `ru`: `Прокрутить вверх`
   - If a visible text label is used instead of icon-only UI, add short labels as well.

## Design Direction
- Prefer a minimal floating action button rather than a full-width or inline control.
- Keep it hidden at the top of the page.
- Make it feel native to the existing clean card-based interface.
- Use either a simple arrow glyph or a short label, but avoid introducing a heavier visual pattern than the rest of the app.

## Validation Plan
1. Run `npm run lint`.
2. Run `npm run build`.
3. Manual checks:
   - On `Home`, scroll down and confirm the button appears only after the threshold.
   - On category pages, confirm the button behaves the same way.
   - Click the button and verify the page scrolls back to the top.
   - Check mobile width to ensure no overlap issues with the sticky header and sidebar.
   - Confirm localization updates correctly for `en`, `ua`, and `ru`.
   - Confirm keyboard focus remains visible.

## Files Expected To Change
- `src/App.jsx`
- `src/i18n.js`
- `src/components/ScrollToTop/ScrollToTop.jsx`
- `src/components/ScrollToTop/ScrollToTop.module.css`
