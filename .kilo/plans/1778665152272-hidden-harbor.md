# План исправления смещения сайдбара при скролле

## Что удалось выяснить
- Основной кандидат на регрессию находится в `src/components/Sidebar/Sidebar.module.css`.
- На десктопе сайдбар использует `position: sticky`, что само по себе выглядит корректно.
- На мобильных сайдбар переключается в `position: fixed` и сейчас завязан на жёсткие значения `65px` и `height: calc(100dvh - 65px)`.
- `100dvh` может пересчитываться во время скролла на мобильных браузерах из-за скрытия/показа browser chrome, из-за чего панель визуально "плавает".
- Дополнительно offset сайдбара опирается на CSS-переменную `--header-height`, но в mobile-ветке часть значений всё ещё захардкожена, что повышает риск мелких сдвигов.

## План изменений
1. Обновить `src/components/Sidebar/Sidebar.module.css`.
   - Заменить мобильные хардкоды `65px` на `var(--header-height)`.
   - Убрать зависимость mobile sidebar от `height: calc(100dvh - 65px)`.
   - Перевести mobile sidebar на схему с `top: var(--header-height)` и `bottom: 0`, чтобы высота фиксировалась без скачков viewport unit при скролле.
   - Аналогично обновить backdrop (`inset`) на использование `var(--header-height)`.

2. Проверить `src/App.module.css`.
   - Убедиться, что `--header-height` остаётся единым источником правды для desktop/mobile offset.
   - При необходимости оставить существующие значения, но использовать их консистентно во всех связанных правилах сайдбара.

3. При необходимости стабилизировать header.
   - Если после CSS-правки останется заметный микросдвиг, проверить `src/components/Header/Header.module.css` и при необходимости зафиксировать `min-height`/`height`, чтобы реальная высота header точнее соответствовала `--header-height`.
   - Это делать только если базовое исправление в `Sidebar.module.css` не закрывает проблему.

## Проверка после правок
1. Запустить `npm run lint`.
2. Запустить `npm run build`.
3. Визуально проверить:
   - desktop: сайдбар не дёргается и ведёт себя стабильно при вертикальном скролле;
   - mobile/tablet (`<= 900px`): drawer не смещается при скролле страницы и не плавает при изменении высоты viewport.

## Ожидаемый результат
- Сайдбар перестанет слегка смещаться при скролле.
- Mobile drawer будет позиционироваться стабильно относительно header.
- Логика offset станет проще и менее чувствительной к пересчёту viewport height.
