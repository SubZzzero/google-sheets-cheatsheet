# План: светлая/тёмная тема

## Цель
Аккуратно внедрить полноценную светлую и тёмную тему без ломки текущей структуры приложения, сохранив текущий визуальный стиль, адаптивность и доступность.

## Что есть сейчас
- Тема только светлая.
- Цвета почти полностью захардкожены в CSS Modules.
- Верхний уровень состояния уже живёт в `src/App.jsx`, поэтому тема логично добавляется туда же.
- Для пользовательских настроек уже есть паттерн с `localStorage` (сейчас используется для `locale`).
- Самая удобная точка для UI-переключателя: `src/components/Header/Header.jsx`.

## Рекомендуемый подход
1. Добавить состояние темы в `src/App.jsx`.
2. Инициализировать тему так:
   - сначала сохранённое значение из `localStorage`,
   - если его нет — системная настройка `prefers-color-scheme`,
   - запасной вариант — `light`.
3. Синхронизировать тему с `document.documentElement` через `data-theme` и динамический `color-scheme`.
4. Перенести общие цвета в CSS custom properties в `src/assets/style.css`.
5. Заменить хардкоженные цвета в модулях компонентов на токены.
6. Добавить компактный переключатель темы в `Header` рядом с переключателем языка.
7. Добавить раннюю установку темы в `index.html`, чтобы избежать вспышки светлой темы при первом рендере.

## Шаги реализации
1. В `src/App.jsx` добавить:
   - `theme` state,
   - lazy initializer для чтения сохранённой/системной темы,
   - `useEffect` для записи `data-theme`, `color-scheme` и сохранения выбора.
2. В `src/i18n.js` добавить локализованные строки для переключателя темы:
   - подпись/aria-label,
   - текст для светлой темы,
   - текст для тёмной темы.
3. В `src/components/Header/Header.jsx` прокинуть тему и обработчик переключения.
4. В `Header.module.css` добавить стили компактного theme toggle так, чтобы он не ломал текущую вёрстку на `900px` и `640px`.
5. В `src/assets/style.css` завести корневые токены, например:
   - фон страницы,
   - основной/вторичный текст,
   - surface/card background,
   - border,
   - muted background,
   - accent,
   - focus outline,
   - overlay/backdrop,
   - shadow.
6. Добавить блоки `:root` и `[data-theme="dark"]` с двумя палитрами.
7. По всем CSS Modules заменить прямые `#...` и `rgba(...)` на переменные, сохранив текущую иерархию поверхностей:
   - page background,
   - header/footer,
   - карточки,
   - hero/gradient blocks,
   - code/example blocks,
   - hover/active/focus states,
   - sidebar backdrop,
   - floating button.
8. В `index.html` добавить маленький inline-скрипт перед монтированием React, который заранее ставит `data-theme` по `localStorage` или системной настройке.
9. Проверить, не теряются ли `logo.png` и `github.png` на тёмном фоне; при необходимости скорректировать фон контейнера или добавить мягкий контрастный фон/фильтр только вокруг иконки, без замены ассетов.

## Дизайн-ограничения
- Не менять структуру экранов и навигацию.
- Не вводить новый глобальный стор или сложную абстракцию.
- Не смешивать тему с данными формул.
- Сохранить текущий зелёно-серый характер интерфейса, а не делать полностью новую визуальную систему.

## Основные риски
- Почти все цвета сейчас локальные, поэтому легко пропустить hover/focus/overlay состояния.
- Без ранней установки темы в `index.html` возможен flash светлой темы.
- В `Header` мало места на мобильных ширинах, поэтому переключатель должен быть очень компактным.
- Нативные элементы и скроллбары останутся светлыми, если не обновлять `color-scheme`.

## Проверка после реализации
1. `npm run lint`
2. `npm run build`
3. Ручная проверка:
   - desktop/mobile,
   - home screen,
   - category screen,
   - mobile sidebar,
   - copy button states,
   - scroll-to-top button,
   - focus-visible и читаемость текста в обеих темах.

## Подтверждённое поведение
Тема при первом открытии берётся из системной настройки, после ручного переключения выбор пользователя сохраняется.
